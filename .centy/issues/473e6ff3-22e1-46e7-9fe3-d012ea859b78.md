# Nav buttons 11 & 12 appear as double nested boxes in TUI

## Description

Buttons 11 and 12 (back/forward navigation) in the TUI display appear to have a double-box / nested-box visual artifact.

## Root Cause

Two separate issues:

### 1. Double box appearance on nav buttons
`render_button` in `src/presentation/tui/render.rs` uses `Block::bordered().title(format!(" {key} "))` for all buttons, embedding the button number in the top border line (e.g. `┌─ 11 ─────┐`). For nav buttons, a single centered line of text ("◄ back" / "fwd ►") sits in otherwise empty inner space, making the titled border look like a header above an inner sub-container — giving the impression of nested boxes. Action buttons (e.g. button 2 "Terminal") don't exhibit this because 3 lines of text fill the inner area uniformly.

### 2. println! bleed-through into alternate screen
`main.rs:48` calls `println!("[init] listening — press keys (11=back, 12=forward)");` after spawning the TUI thread. Since both threads share the same stdout fd, this print can land inside the alternate screen, showing as `eys (11=back, 12=forward)` at the bottom of the TUI.

## Files
- `src/presentation/tui/render.rs` — nav button rendering (lines 65–89)
- `src/main.rs:48` — println! that bleeds through
