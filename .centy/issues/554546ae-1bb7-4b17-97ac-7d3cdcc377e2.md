# Add persistent structured logging

## Problem

The app currently has no persistent logging. All output is scattered `println!`/`eprintln!` calls and a 10-item in-memory TUI log. When something goes wrong (USB disconnect, bad state, crash), there is zero forensic data — the TUI log scrolls off and stdout/stderr are lost.

## Proposed Solution

Adopt `tracing` + `tracing-subscriber` as the logging backend, with dual output:

1. **File log** — persistent, timestamped, full history at `~/.config/vibe-keyboard/app.log`
2. **TUI in-memory log** — keep the existing live feed in the UI, but drive it from the same logging system via a custom subscriber layer

## Changes

- Add `tracing` and `tracing-subscriber` (with `RollingFileAppender`) to `Cargo.toml`
- Initialize the subscriber in `main.rs` before any device setup
- Replace all `println!`/`eprintln!` with `info!`, `warn!`, `error!` macros
- Wire `AppState`'s event log to receive tracing events via a custom `Layer`, so the TUI display stays in sync automatically
- Support `RUST_LOG` env var for filtering

## Benefits

- Crash/disconnect forensics — full log on disk
- Timestamps and log levels on every message
- `RUST_LOG=debug` for deep debugging without code changes
- No more scattered ad-hoc `[module]` prefix convention — tracing handles module paths
- Single source of truth for both file and TUI log
