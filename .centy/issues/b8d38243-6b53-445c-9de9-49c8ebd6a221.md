# Replace custom spinner logic with throbber-widgets-tui library

## Context

The current spinner implementation in `src/presentation/tui/render.rs` uses a hand-rolled approach:
- A `const SPINNER: [&str; 8]` array of braille characters
- Time-based frame selection using `SystemTime::now()` to index into the array

## Task

Replace the custom spinner with the [`throbber-widgets-tui`](https://crates.io/crates/throbber-widgets-tui) crate, which is a ratatui-native spinner widget.

## Changes needed

1. Add `throbber-widgets-tui` to `Cargo.toml`
2. Add `ThrobberState` to `AppState`
3. Tick the state on each render/event loop cycle
4. Replace the manual spinner render block in `render_button` with the throbber widget

## Trade-offs to consider

The current implementation is stateless (no extra state needed). The library approach requires:
- `ThrobberState` in `AppState`
- Calling `.calc_next()` on each tick

Gains: access to built-in spinner styles, cleaner widget API, no manual time math.
