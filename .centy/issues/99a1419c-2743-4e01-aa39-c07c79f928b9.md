# Add issue action screen: open in VSCode, web, or centy workspace

## Problem

When pressing an issue key in the `CentyIssueList` screen, nothing happens — the key press is unhandled. There is no way to act on a selected issue from the keyboard UI.

## Proposed Solution

Add a `CentyIssueActions` screen (parallel to the existing `CentyProjectActions` screen) that is pushed onto the navigation stack when an issue is selected. It exposes three action buttons:

1. **Open in VSCode** — opens the issue's `.md` file in VS Code via `code <file_path>`
2. **Open in web** — opens the issue URL (`https://app.centy.io/{org}/{project}/issues/{id}`) in the default browser via `open <url>`
3. **Open centy workspace** — launches the centy cockpit for the project via `pnpm dlx centy cockpit`

## Changes

- **`domain/actions.rs`** — add `id: String` and `file_path: Option<String>` to `CentyIssue`; add `CentyIssueActions { issue_number: u64, project_name: String }` variant to `ScreenView`
- **`domain/navigation.rs`** — add `CentyIssueActions { issue: CentyIssue, project_name: String }` variant to `Screen`; update `back()` / `out()` to handle the new screen (pops to `CentyIssueList`)
- **`application/pages/centy.rs`** — extend `parse_centy_issues_json` to extract `id` (UUID slug) and `file_path` (from `filePath` / `path` JSON fields) for each issue
- **`application/pages/mod.rs`** — handle key presses 1–10 on `CentyIssueList` to push `CentyIssueActions` for the selected issue; add `render_screen` branch for `CentyIssueActions` showing three labelled action buttons; implement action handlers for the three buttons
- **`application/mod.rs`** — route `CentyIssueActions` key events to the three action functions
