# Add persistent global device state via TOML file

## Summary

Add a `src/state.rs` module that manages global device state, persisted to `~/.config/vibe-keyboard/state.toml`.

## Goals

- Define a `DeviceState` struct with `serde::Serialize` / `Deserialize`
- Load state from TOML on startup (defaults if file missing)
- Save state back when it changes
- Extensible for future fields (button mappings, macros, etc.)

## Initial fields

```toml
current_page = 0
brightness = 25
```

## Dependencies to add

```toml
serde = { version = "1", features = ["derive"] }
toml = "0.8"
```

## Notes

- State file: `~/.config/vibe-keyboard/state.toml`
- Gracefully handle missing file (use defaults, write on first run)
