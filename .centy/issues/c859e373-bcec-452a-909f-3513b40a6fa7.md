# Add open-log-file action to page 2 button 14

## Problem

The app has no way to quickly access the log file from the keyboard UI. The TUI log panel has been removed in favour of a persistent file log at `~/.config/vibe-keyboard/app.log`.

## Proposed Solution

Add a button action on page 2 (index 1), key 14 that opens the log file in VS Code when pressed.

## Changes

- Add `open_log_file()` function in `application/pages.rs` that calls `code ~/.config/vibe-keyboard/app.log`
- Register key 14 in `page_actions` for page 1 with name/title/description
- Handle key 14 press in `handle_key_event` under page 1 branch
- Generate a log-file button image in `infrastructure/images/log_file.rs`
- Export image and call `send_button_image` in `activate_page` for page 1
